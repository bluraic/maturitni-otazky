Otázka 16 – Definice pojmů z oblasti webu

1. WWW (World Wide Web)

World Wide Web (WWW) je služba internetu, která umožňuje zobrazování a propojování hypertextových dokumentů – tzv. webových stránek. Tyto stránky mohou obsahovat text, multimédia a interaktivní prvky.

Důležité upozornění: WWW není totéž co internet. Internet je celosvětová hardwarová síť (infrastruktura), zatímco WWW je pouze jedna z mnoha služeb, které na této síti běží (podobně jako e-mail nebo FTP).

Jak WWW funguje (Model Klient–Server)

  • Klient (Prohlížeč): Uživatel zadá adresu, prohlížeč vytvoří požadavek (request) přes protokol HTTP/HTTPS.
  • Server: Webový server požadavek zpracuje a pošle zpět odpověď (response) — obvykle soubory HTML, CSS a JS.
  • Vykreslení: Prohlížeč tyto soubory interpretuje a zobrazí výslednou stránku.

URL — adresa webové stránky

Každý zdroj na webu má svou unikátní adresu URL (Uniform Resource Locator).

https://www.example.com/sekce/stranka.html

  • https:// – Protokol (zabezpečený).
  • www.example.com – Doména (adresa serveru).
  • /sekce/stranka.html – Cesta k souboru na serveru.

2. HTML (HyperText Markup Language)

HTML je standardizovaný značkovací jazyk (nikoliv programovací). Definuje strukturu a sémantiku webového dokumentu pomocí tzv. tagů (značek).

<!DOCTYPE html>
<html lang="cs">
<head>
    <meta charset="UTF-8">
    <title>Název stránky</title>
</head>
<body>
    <h1>Nadpis stránky</h1>
    <p>Toto je odstavec textu.</p>
</body>
</html>

3. XHTML (Extensible HyperText Markup Language)

XHTML je přísnější verze HTML založená na XML. Vyžaduje absolutní čistotu kódu, což mělo zajistit lepší strojové zpracování dat.

  • Pravidla: Všechny tagy musí být uzavřené (např. <br />), psané malými písmeny a atributy musí být vždy v uvozovkách.
  • Současnost: Dnes je XHTML vytlačeno HTML5, které je flexibilnější a obsahuje moderní prvky pro video, audio a interaktivitu.

4. Domény a DNS

Doména je lidsky čitelný název pro IP adresu serveru. Abychom si nemuseli pamatovat čísla jako 142.250.184.206, píšeme google.com.

Struktura domény

subdoména.doména2řádu.TLD (např. blog.seznam.cz)

DNS (Domain Name System)

DNS funguje jako „telefonní seznam internetu“. Když zadáte doménu, prohlížeč se zeptá DNS serveru na odpovídající číselnou IP adresu, aby věděl, ke kterému počítači se má připojit.

5. TLD — domény nejvyšší úrovně

TLD (Top Level Domain) je koncovka domény, která určuje její příslušnost nebo účel.

Typ TLD Příklad Význam
ccTLD (národní) .cz, .sk, .de Přiřazeny konkrétním státům.
gTLD (generické) .com, .org, .net Původně pro komerční či neziskové org.
new gTLD (moderní) .shop, .app, .blog Tématické domény pro lepší identifikaci.

  • Dokážete vysvětlit rozdíl mezi statickým a dynamickým webem?
  • Proč je dnes protokol HTTPS naprostým standardem oproti starému HTTP?
  • Co se stane, když zadám adresu, která neexistuje? (Chybový kód 404).

Otázka 17 – Základní syntaxe HTML, CSS, JavaScriptu a PHP

Moderní webová stránka je tvořena kombinací těchto čtyř technologií. HTML tvoří kostru, CSS vzhled, JavaScript zajišťuje interaktivitu na straně uživatele a PHP logiku na straně serveru.

1. HTML – HyperText Markup Language

HTML je značkovací jazyk. Jeho syntaxe je založena na tagy (značkách), které jsou obvykle párové.

Základní kostra dokumentu

<!DOCTYPE html>
<html lang="cs">
<head>
    <meta charset="UTF-8">
    <title>Název stránky</title>
</head>
<body>
    <h1>Hlavní nadpis</h1>
    <p>Text odstavce s <a href="#">odkazem</a>.</p>
</body>
</html>

Časté HTML značky

Značka Význam
<h1> – <h6> Nadpisy různých úrovní důležitosti.
<p> Odstavec souvislého textu.
<img src="..."> Vložení obrázku (nepárový tag).
<div> Univerzální blokový prvek pro strukturu.

2. CSS – Cascading Style Sheets

CSS používá systém selektorů a vlastností. „Kaskádové“ znamená, že se styly mohou vrstvit a dědit.

Ukázka syntaxe

/* Selektor vybere prvek, složené závorky obsahují pravidla */
body {
    background-color: #f0f0f0; /* barva pozadí */
    font-family: Arial, sans-serif;
}

.zvyrazneny-text {
    color: red;
    font-weight: bold;
}

Typy selektorů

  • Typový: p { ... } (ovlivní všechny odstavce).
  • Třída: .moje-trida { ... } (pro více prvků, v HTML jako class="...").
  • ID: #unikatni { ... } (pro jeden konkrétní prvek na stránce).

3. JavaScript (JS)

JavaScript je programovací jazyk, který běží v prohlížeči (client-side). Umožňuje manipulovat s obsahem stránky bez jejího znovunačtení pomocí DOM (Document Object Model).

Základní logika a události

// Definice proměnných a funkce
let pocetKliknuti = 0;

function pozdrav() {
    pocetKliknuti++;
    alert("Klikli jste už " + pocetKliknuti + "x!");
}

V HTML pak na JS reagujeme pomocí eventů, např.: <button onclick="pozdrav()">Tlačítko</button>.

4. PHP – Hypertext Preprocessor

PHP je skriptovací jazyk, který běží na serveru. Klient (prohlížeč) PHP kód nikdy neuvidí, dostane pouze výsledné HTML, které PHP vygenerovalo.

Základní syntaxe

PHP kód se vkládá do speciálních značek <?php ... ?>. Proměnné vždy začínají symbolem $.

<?php
$jmeno = "Admin";
$hodina = date("H");

if ($hodina < 12) {
    echo "Dobré ráno, " . $jmeno;
} else {
    echo "Dobrý den, " . $jmeno;
}
?>

Hlavní rozdíl: JS vs. PHP

  • JavaScript: Vidí uživatel v kódu, provádí se u něj v počítači (např. kontrola, zda je v poli e-mailu zavináč).
  • PHP: Skryté na serveru, pracuje s databází (např. ověření, zda je heslo uživatele správné).

Závěr

Pamatujte, že HTML je podstatné jméno (věc), CSS je přídavné jméno (vlastnost), JavaScript je sloveso (akce) a PHP je mozek v zákulisí (server).

Otázka 18 – Webové stránky: Typy, layout a sémantika

1. Statické a dynamické webové stránky

Základní rozdíl spočívá v tom, kdy a kde obsah vzniká. Zatímco statický web je "vytesán do kamene", dynamický web se skládá jako skládačka v okamžiku, kdy na něj kliknete.

1.1 Statické webové stránky

  • Princip: Obsah je pevně uložen v souborech .html na serveru.
  • Výhody: Blesková rychlost, vysoká bezpečnost (není tam databáze, kterou by šlo hacknout), levný hosting.
  • Nevýhody: Pro každou změnu (např. v menu) musíte ručně přepsat všechny soubory.

1.2 Dynamické webové stránky

  • Princip: Stránka se generuje až na vyžádání (pomocí PHP, Pythonu apod.). Data se tahají z databáze.
  • Výhody: Snadná správa přes redakční systémy (WordPress), možnost přihlášení, personalizace (e-shopy, sociální sítě).
  • Nevýhody: Pomalejší načítání, vyšší riziko napadení (SQL injection).
Kritérium Statický web Dynamický web
Obsah Pevný (v souboru) Generovaný (z databáze)
Interaktivita Minimální Vysoká (formuláře, uživatelé)
Správa Ruční editace kódu Administrační rozhraní

2. Layout webu a sémantika

Dříve se weby skládaly pomocí neviditelných tabulek (špatně!), dnes používáme sémantické tagy HTML5. Ty říkají prohlížeči a vyhledávačům, co která část stránky znamená.

Významné sémantické tagy:

  • <header> – Hlavička (logo, název).
  • <nav> – Hlavní navigace.
  • <main> – Unikátní hlavní obsah stránky (pouze 1x na stránku).
  • <section> a <article> – Logické bloky a samostatné příspěvky.
  • <aside> – Postranní panel (reklamy, související odkazy).
  • <footer> – Patička (autorská práva, kontakt).

3. Základní HTML tagy v praxi

HTML tagy dělíme na blokové (začínají na novém řádku) a řádkové (jsou součástí textu).

<h1>Hlavní nadpis</h1>
<p>Toto je odstavec s <strong>tučným textem</strong> a <em>kurzívou</em>.</p>
<hr> <!-- Vodorovná čára -->
<a href="https://google.com">Odkaz na Google</a>
<img src="pes.jpg" alt="Fotka zlatého retrívra">

4. Tabulky v HTML

Pozor: Nikdy nepoužívejte tabulky pro tvorbu vzhledu (layoutu)! Slouží výhradně pro výpis dat, jako jsou ceníky nebo rozvrhy.

Struktura tabulky:

<table>
    <thead>
        <tr>
            <th>Předmět</th>
            <th>Známka</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Informatika</td>
            <td>1</td>
        </tr>
    </tbody>
</table>

Závěrečné shrnutí

  • Statický web: Levný a rychlý, ale nepohodlný na údržbu.
  • Sémantika: Pomáhá SEO (vyhledávačům jako Google lépe rozumět webu).
  • Atribut ALT: U obrázků je povinný kvůli přístupnosti (pro nevidomé).

Otázka 19 – Parametry ovlivňující čitelnost textu (HTML + CSS)

1. Co je čitelnost textu (Readability)

Čitelnost je míra toho, jak snadno a pohodlně dokáže čtenář text přečíst a porozumět mu. Na webu je kritická, protože uživatelé text často jen skenují. Pokud narazí na nepřehledný blok textu, web okamžitě opouštějí.

2. Hlavní typografické parametry v CSS

Správné nastavení těchto vlastností v CSS zásadně ovlivňuje únavu očí uživatele.

2.1 Font-family (Typ písma)

Pro web se doporučují bezpatková písma (sans-serif), jako je Arial, Roboto nebo Open Sans, protože jsou čitelnější na obrazovkách s nižším rozlišením.

body { font-family: 'Roboto', Arial, sans-serif; }

2.2 Font-size a Line-height (Velikost a řádkování)

  • Velikost: Pro běžný text se dnes doporučuje 16–18 px.
  • Řádkování (Line-height): Určuje vertikální prostor mezi řádky. Ideální hodnota je 1.5 až 1.8. Pokud je řádkování příliš malé, text se "slévá".

2.3 Délka řádku a šířka kontejneru

Příliš dlouhé řádky unavují oko, protože je těžké najít začátek dalšího řádku. Ideální délka je 50–75 znaků. V CSS toho docílíme pomocí max-width a jednotky ch (šířka znaku '0').

.content { max-width: 70ch; margin: 0 auto; }

2.4 Kontrast a Barvy

Dle metodiky WCAG by měl být minimální kontrastní poměr 4.5:1. Nejlepší je tmavě šedý text na velmi světle šedém nebo bílém pozadí (čistě černá na čistě bílé může působit příliš agresivně).

3. Hierarchie a sémantika

Struktura stránky musí být jasná na první pohled díky správnému použití nadpisů <h1> až <h6>.

Pravidlo: Nadpis <h1> smí být na stránce pouze jeden. Nadpisy nesmí přeskakovat úrovně (např. z H1 rovnou na H3).

4. Box Model: Margin vs. Padding

Prostor kolem textu (tzv. "white space") dává obsahu "vzduch" a zvyšuje přehlednost.

  • Padding: Vnitřní prostor (např. odsazení textu od okraje barevného rámečku).
  • Margin: Vnější prostor (např. mezera mezi dvěma odstavci).

5. Jednotky: Absolutní vs. Relativní

Jednotka Typ Použití
px Absolutní Pevné rozměry (okraje, stíny).
rem / em Relativní Ideální pro fonty (reaguje na nastavení prohlížeče).
% / vw / vh Relativní Responzivní layouty (šířka kontejneru).

6. Přístupnost (Accessibility)

Čitelnost není jen o vzhledu, ale i o tom, aby web přečetly čtečky pro nevidomé:

  • Používání sémantických tagů (<article>, <section>).
  • Alt texty u obrázků.
  • Dostatečný kontrast pro lidi se zrakovými vadami.

Tipy pro zkoušku (15min výklad)

  1. Začněte definicí: Co je to čitelnost a proč na ní záleží (uživatelé neskenují, ale skáčou po textu).
  2. Technické parametry: Vysvětlete trojici font-size, line-height a max-width.
  3. Porovnejte jednotky: Proč je rem lepší než px pro přístupnost.
  4. Vizuální hierarchie: Proč je důležitý kontrast a sémantika nadpisů.
  5. Závěr: Zmiňte responzivitu – čitelnost na mobilu vs. na desktopu.
